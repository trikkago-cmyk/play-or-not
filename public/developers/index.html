玩吗 - Internal Integration Notes

「玩吗」是一个桌游推荐与规则问答应用。它主要帮助用户在聚会前快速选桌游，并在开局后即时解释规则、胜利条件和常见追问。 这个页面主要作为内部集成说明使用，不属于面向普通玩家的产品入口。

桌游推荐 规则讲解 裁判追问 语音转写 RAG 代理

产品能做什么

推荐模式

根据人数、时长、复杂度、场景和氛围推荐合适的桌游。

裁判模式

围绕某个具体游戏回答流程、胜利条件、平局、特殊效果等问题。

语音输入

前端支持录音，后端通过 /api/stt 把音频转成文字后继续聊天。

当前公开边界

  • 当前公开接口主要适合问答、推荐和语音转写，不适合复杂写操作。
  • 当前没有公开的 API key 发放页、OAuth 文档或 service account。
  • 当前没有公开的用户偏好写入 API，也不保证长期会话持久化。
  • 人类登录当前使用邮箱验证码 + 浏览器 session cookie，Agent 更适合走公开接口层。

发现入口

  • /llms.txt：给 LLM / Agent 的简明说明
  • /capabilities.json：机器可读的任务与能力清单
  • /openapi.json：结构化 API schema
  • /api/chat：聊天接口自描述 JSON
  • /api/rag?describe=1：RAG 代理说明
  • /api/stt：语音转写接口说明
  • /api/auth/email/send：邮箱验证码发送说明
  • /api/auth/email/verify：邮箱验证码校验说明
  • /api/auth/session?describe=1：登录态读取说明

最常用接口

POST /api/chat

推荐桌游或解释规则的主入口。请求体兼容 OpenAI 风格的 messages 数组。

{
  "messages": [
    {
      "role": "user",
      "content": "推荐一个适合 4 人破冰、30 分钟内能讲完规则的桌游"
    }
  ],
  "temperature": 0.7,
  "max_tokens": 800
}

POST /api/stt

使用 multipart/form-data 上传音频文件，字段名为 file。

GET /api/rag 与 POST /api/rag

GET 默认返回健康检查；POST 将 JSON 查询代理到 Python RAG 服务。

邮箱验证码登录接口

人类登录通过 POST /api/auth/email/send 发送验证码， 再通过 POST /api/auth/email/verify 建立浏览器 session， 最后可通过 GET /api/auth/session 读取当前登录状态。

任务与路由映射

  • recommend_game：优先走 POST /api/chat，可选传 "task":"recommend_game"
  • explain_rules：优先走 POST /api/chat，适合规则、胜利条件、平局问答
  • referee_followup：优先走 POST /api/chat，建议把近期对话历史一起带上
  • transcribe_audio：走 POST /api/stt，用 multipart/form-data 上传 file
  • query_rag：仅在需要下层知识库代理时再走 POST /api/rag

如果你想直接获取这份映射的 JSON 版本，读取 /capabilities.json 即可。

推荐的 Agent 调用顺序

  1. 先读取 /llms.txt 或 /developers/ 建立整体认知。
  2. 再读取 /openapi.json 或接口自描述 JSON。
  3. 优先从 POST /api/chat 完成推荐与规则问答主任务。
  4. 如果需要语音转写，使用 POST /api/stt。
  5. 如果需要走底层知识库或健康检查，再访问 /api/rag。
